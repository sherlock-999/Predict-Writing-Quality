# Linking Writing Process to Writing Quality

Kaggle competition: predict a holistic essay score (0.5 – 6.0) from keystroke log data.

## Project Structure

```
writing_process/
├── data/                       # raw competition data (not tracked)
│   ├── train_logs.csv
│   ├── train_scores.csv
│   ├── test_logs.csv
│   └── sample_submission.csv
├── preprocess/                 # feature engineering modules
│   ├── __init__.py
│   ├── essay_reconstruction.py # replay keystroke events to recover final text
│   ├── keystroke_stats.py      # count, timing, word-length features from raw logs
│   ├── burst_stats.py          # production & revision burst lengths, idle gaps
│   ├── text_stats.py           # word/sentence/paragraph stats from reconstructed text
│   └── efficiency.py           # product-to-keys ratio, typing speed
├── models/                     # saved fold models (not tracked, generated by train_lgbm.py)
├── eda_plots/                  # output plots from eda.py
├── lgbm_plots/                 # output plots from train_lgbm.py
├── eda.py                      # exploratory data analysis + feature correlation plots
├── important_features.py       # 120-feature pipeline (gain >= 15) + category definitions
├── train_lgbm.py               # 5-fold CV training + feature importance plots
├── predict.py                  # inference on test set → submission.csv
├── requirements.txt
├── .gitignore
└── README.md
```

## Setup

```bash
conda create -n exp python=3.11
conda activate exp
pip install -r requirements.txt
```

## Usage

Run in order:

```bash
# 1. EDA — understand the data and feature correlations
conda run -n exp python eda.py

# 2. Train — 5-fold CV, saves fold models to models/
conda run -n exp python train_lgbm.py

# 3. Predict — generates submission.csv from test_logs.csv
conda run -n exp python predict.py
```

## Features (120 Total)

All features are computed per essay from the raw keystroke log. Feature engineering is modular,
organized across 8 conceptual categories:

| Category | Examples | Module |
|---|---|---|
| **Word Characteristics** | word length (mean, max, quantiles), word count stats | keystroke_stats, text_stats |
| **Sentence Characteristics** | sentence length, words per sentence, sentence count | text_stats |
| **Paragraph Characteristics** | paragraph length, words per paragraph, paragraph count | text_stats |
| **Punctuation & Formatting** | comma, period, newline, shift, caps-lock counts | keystroke_stats |
| **Typing Speed & Fluency** | keys per second, action time stats, key diversity | keystroke_stats |
| **Pausing Behavior** | idle gaps, pause counts at various thresholds (0.5s–3s bands) | burst_stats |
| **Revision Behavior** | deletion counts, revision burst lengths | keystroke_stats, burst_stats |
| **Production Flow & Navigation** | production burst lengths, cursor movement, session timing, efficiency | burst_stats, keystroke_stats, efficiency |

### Feature Computation Pipeline

The pipeline in `important_features.py` combines modular preprocessing stages:

1. **count_features** — count activities, keys, text changes
2. **input_word_features** — word-length distribution from typed q-sequences
3. **timing_features** — action_time, down_time, up_time, cursor_position, word_count aggregations
4. **idle_features** — inter-key idle gaps and pause-count thresholds
5. **p_burst_features** — production burst lengths (runs < 2s apart)
6. **r_burst_features** — revision burst lengths (consecutive deletions)
7. **reconstruct_essay** — replay keystroke events to recover final essay text
8. **word_features** — word length stats from reconstructed text
9. **sentence_features** — sentence length and word-count stats
10. **paragraph_features** — paragraph length and word-count stats
11. **efficiency_features** — product-to-keys ratio, keys-per-second

> **Note**: All alphabetic characters in `text_change` are anonymised as `q`.
> Punctuation (`.`, `,`, `?`, `'`, `-`, `\n`) and formatting keys are preserved.

## Model

**LightGBM regression** with 5-fold stratified cross-validation.

- **Stratification**: on essay score bins (ensures each fold has representative score distribution)
- **Ensemble**: final test predictions are the mean of all 5 fold models
- **Output range**: clipped to [0.5, 6.0] (valid score range)

### Key Hyperparameters

```
objective:           'regression'

lambda_l1:           0.1
lambda_l2:           0.1
n_estimators:        2000
early_stopping_rounds: 50
```
